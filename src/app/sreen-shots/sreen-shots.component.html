Screen Shots

This is the login page of the application. Only administrators have access to log in.

This is the dashboard of our application. Here, we feature a live map alongside a table that highlights the most recent events detected by the system. Additionally, we provide an equipment status table for all cameras positioned at various locations.

Each event in the Latest Event table is initially labeled with a default status of "Unseen." When an administrator reviews an event and decides to take action, they can click on the "Seen" button linked with that particular event. This action triggers a confirmation popup box to verify the change of status.

Upon clicking, the event's status will change from "Unseen" to "Seen".

When a new Fire event is detected and appears in the Latest Event table, the data associated with that event should be displayed in highlighted rows to facilitate rapid identification.

We also include an image for each specific event in the Latest Event table. This enhances the visibility and comprehension of the events listed.

This is the live streaming screen, where live feeds from each camera located at various locations are displayed in real-time.

Here, we present an example of an event table encompassing all recorded events. This particular table pertains to Object Detection Events. We comprehensively display all relevant data related to object detection events, including accompanying images and videos for each specific event.
